Merge Sort (병합 정렬)

병합 정렬(Merge Sort)은 재귀 호출(Recursive Call)을 이용하여 정렬하는 알고리즘입니다.
병합 정렬은 리스트를 두 그룹으로 나누어 각각을 정렬하고 그 정렬된 두 그룹을 합하는 방식입니다.
나뉘어진 그룹은 내부적으로 다시 두 그룹으로 나누고 각각을 정렬한 후 정렬된 두 그룹을 합합니다.
이러한 방식을 재귀(Recursive) 방식이라고 합니다.

리스트를 쪼개어서 무엇인가 수행을 하는 방식은 분할 정복(Divide and Conquer)이라고 하는데, 병합 정렬이 바로 재귀적으로 분할 정복(Divide and conquer recursively)을 하는 경우라 할 수 있겠습니다.

문제


list = [35, 9, 2, 85, 17];

위의 리스트 안의 자료를 작은 수부터 큰 수 순서로 배열하는 정렬 알고리즘을 만드시오.

[언뜻 생각한 알고리즘]

step 1: 리스트의 가운데인 2를 중심으로 두 그룹으로 나눕니다.
        -> g1 = [35, 9, 2], g2 = [85, 17]

(* 리스트 요소의 개수가 홀수개이면 한 그룹은 다른 그룹보다 개수가 하나 많은 그룹이 될 것입니다. 문제될 것은 없습니다.)

step 2: g1 그룹을 가운데인 9를 중심으로 두 그룹으로 나누고, g2 그룹을 두 그룹으로 나눕니다.
        -> g1_1 = [35, 9], g1_2 = [2], g2_1 = [85], g2_2 = [17]

step 3: g1_1 그룹을 두 그룹으로 나눕니다. g1_2 그룹과 g2 그룹은 이미 더 이상 나눌 수 없는 하위 그룹들로 나뉘었기 때문에 별다른 것을 수행하지 않습니다.
        -> g1_1_1 = [35], g1_1_2 = [9], g1_2 = [2], g2_1 = [85], g2_2 = [17]

** 리스트를 모두 divide 하였습니다.

step 4: g1_1_1 의 값(35)과 g1_1_2 의 값(9)을 비교하여 순서를 정렬하고 한 그룹으로 묶습니다.
        -> g1_1 = [9, 35], g1_2 = [2], g2_1 = [85], g2_2 = [17]

step 5: g1_1 의 값(9, 35)과 g1_2 의 값(2)을 비교하여 순서를 정렬하고 한 그룹으로 묶습니다.
        마찬가지로, g2_1의 값(85)과 g2_2의 값(17)을 비교하여 순서를 정렬하고 한 그룹으로 묶습니다.
        -> g1 = [2, 9, 35], g2 = [17, 85]

step 6: g1 의 값(2, 9, 35)과 g2 의 값(17, 85)을 비교하여 순서를 정렬하고 한 그룹으로 묶습니다.
        -> g = [2, 9, 17, 35, 85]

(이렇게 구구절절 설명하려니 혼란스러우면서도 이런 과정의 설명이 의미가 있나 싶기도 하네요.)

(1) 계속 작은 그룹으로 분할하여 (2) 값들을 비교한 후 (3) 정렬된 그룹을 만들어 나가는 과정은 위의 [그림 1]로도 이해하실 수 있을 것이라 생각합니다.

Time Complexity (복잡도)

분할 정복(Divide and Conquer)은 잘 활용하면 계산 복잡도가 더 낮은 효율적인 알고리즘을 만드는 데 도움이 된다고 합니다.
분할 정복을 이용한 계산 복잡도는 O(n·log n)으로 선택 정렬(Selection Sort)이나 삽입 정렬(Insertion Sort)의 계산 복잡도 O(n²)보다 낮습니다.
