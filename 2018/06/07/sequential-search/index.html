Sequential Search (순차적 탐색)

순차적 탐색(Sequential Search)은 말 그대로 리스트 안에 있는 원소를 하나씩 순차적으로 비교하면서 탐색하는 방법입니다.

예로 아래와 같은 문제가 있습니다.

문제


list = [17, 92, 18, 33, 58, 5, 33, 42];

위의 리스트에서 특정값 33, 900을 찾아서 해당 위치 번호를 반환하시오.
찾고자 하는 값이 없다면 -1을 반환하시오.

[리스트에서 33 찾기]

아래와 같은 과정으로 리스트에서 33을 찾게 될 것입니다.

step 1: 17은 33입니까? -> 아니오 -> 리스트의 다음 값을 비교합니다.
step 2: 92는 33입니까? -> 아니오 -> 리스트의 다음 값을 비교합니다.
step 3: 18은 33입니까? -> 아니오 -> 리스트의 다음 값을 비교합니다.
step 4: 33은 33입니까? -> 예 -> 탐색을 중단하고 해당값의 index 를 반환합니다.

따라서 반환값은 3이 됩니다. (33의 위치 번호(index)가 3)

사실 33은 리스트에 두 개 존재하지만 가장 처음 만난 값의 index 를 반환하는 것으로 가정합니다.



[리스트에서 900 찾기]

아래와 같은 과정으로 리스트에서 900을 찾게 될 것입니다.

step 1: 17은 900입니까? -> 아니오 -> 리스트의 다음 값을 비교합니다.
step 2: 92는 900입니까? -> 아니오 -> 리스트의 다음 값을 비교합니다.
step 3: 18은 900입니까? -> 아니오 -> 리스트의 다음 값을 비교합니다.
…
step 8: 42는 900입니까? -> 아니오 -> 리스트의 마지막까지 비교했으나 찾고자 하는 값은 없습니다.

따라서 반환값은 -1이 됩니다.

Time Complexity (복잡도)

앞서의 33을 찾는 예에서, 만약 33이 리스트의 가장 첫 요소라면 탐색을 1회 하게 될 것입니다.
만약 33이 리스트의 두 번째 요소라면 2회를 탐색하게 될 것입니다.

33은 리스트에서 네 번째 요소이기 때문에 총 네 번을 탐색하여 그 위치를 찾아내었습니다.

만약, 33이 리스트의 가장 마지막 요소라면 리스트 길이만큼인 8회를 탐색하여 위치를 찾아내게 될 것입니다.

즉, 최선의 경우에는 1회 탐색으로 원하는 요소를 찾을 수 있지만, 최악의 경우에는 n회 탐색으로 원하는 요소를 찾아내게 됩니다.

하여 복잡도는 O(n)으로 나타낼 수 있겠습니다.
