Insertion Sort (삽입 정렬)

삽입 정렬(Insertion Sort)은 리스트의 모든 요소를 앞에서부터 차례대로 이미 정렬된 부분과 비교하여 자신의 위치를 찾아 삽입하는 정렬 알고리즘입니다.
리스트의 n번 째 요소를 비교할 차례라고 할 때, n번 째 요소의 앞 부분은 이미 정렬된 상태입니다.
이 정렬된 상태의 요소들과 n번 째 요소를 비교하여 n번 째 요소가 자리할 위치를 찾아 삽입해 주는 방법입니다.

선택 정렬(Selection Sort)의 예와 같은 리스트를 삽입 정렬(Insertion Sort)로 정렬해 보겠습니다.

문제

list = [35, 9, 2, 85, 17];

위의 리스트 안의 자료를 작은 수부터 큰 수 순서로 배열하는 정렬 알고리즘을 만드시오.

[언뜻 생각한 알고리즘]

* 리스트의 두 번째 요소인 9부터 리스트에서의 위치 탐색을 시작합니다.
첫 번째 요소는 비교할 앞선 요소가 없기 때문에 이미 정렬되었다고 간주합니다.

* 정렬이 완료된 것은 초록색, 정렬 대상인 요소는 파란색으로 표시하도록 하겠습니다.

step 0: 35는 첫 번째 요소로 이미 정렬되었다고 간주합니다.
        -> list = [35, 9, 2, 85, 17]

step 1: 9를 9의 위치보다 앞선 요소들(35)과 비교하여 적절한 위치를 찾아 삽입합니다.
        -> list = [9, 35, 2, 85, 17]

step 2: 2를 2위 위치보다 앞선 요소들(9, 35)과 비교하여 적절한 위치를 찾아 삽입합니다.
        -> list = [2, 9, 35, 85, 17]

step 3: 85를 85보다 앞선 요소들(2, 9, 35)과 비교하여 적절한 위치를 찾아 삽입합니다.
        -> list = [2, 9, 35, 85, 17]

step 4: 17을 17보다 앞선 요소들(2, 9, 35, 85)과 비교하여 적절한 위치를 찾아 삽입합니다.
        -> list = [2, 9, 17, 35, 85]

[step 4를 조금 자세히 살펴 보겠습니다.]

step 4-1: 2는 17보다 큰가? -> 아니오 : list = [2, 9, 35, 85, 17]
step 4-2: 9는 17보다 큰가? -> 아니오 : list = [2, 9, 35, 85, 17]
step 4-3: 35는 17보다 큰가? -> 예 : list = [2, 9, 17, 35, 85]

여기서 탐색을 중단합니다. 왜냐하면, 17과 비교할 다른 요소들(35, 85)은 이미 정렬이 된 상태이기 때문에 35와 비교하여 17이 더 작다면 35 이후의 값은 이미 17보다 큰 상태이므로 더 이상 비교는 의미가 없습니다.

Time Complexity (복잡도)

리스트의 첫 요소부터 마지막 요소까지 순차적으로, 해당 요소와 해당 요소의 앞 요소들을 비교하는 방법으로 구성되므로 복잡도는 O(n²)으로 나타낼 수 있겠습니다.
