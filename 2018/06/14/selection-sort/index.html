Selection Sort (선택 정렬)

정렬(Sort)은 알고리즘의 꽃이라고 할 수 있는데, 자료를 크기 순서대로 맞춰 일렬로 나열하는 것을 말합니다.
사전은 단어를 가나다순 혹은 알파벳순으로 나열하는데 이것은 정렬의 굉장히 좋은 예라고 할 수 있습니다.

예로 아래와 같은 문제가 있습니다.

문제

list = [35, 9, 2, 85, 17];

위의 리스트 안의 자료를 작은 수부터 큰 수 순서로 배열하는 정렬 알고리즘을 만드시오.

[언뜻 생각한 알고리즘]

step 0: 결과를 담아 둘 list 를 생성합니다.
        -> result = []

step 1: list의 다섯 요소 중에서 가장 작은 수를 찾아 결과 배열로 이동합니다.
        -> list = [35, 9, 85, 17], result = [2]

step 2: list의 네 요소 중에서 가장 작은 수를 찾아 결과 배열로 이동합니다.
        -> list = [35, 85, 17], result = [2, 9]

step 3: list의 세 요소 중에서 가장 작은 수를 찾아 결과 배열로 이동합니다.
        -> list = [35, 85], result = [2, 9, 17]

step 4: list의 두 요소 중에서 가장 작은 수를 찾아 결과 배열로 이동합니다.
        -> list = [85], result = [2, 9, 17, 35]

step 5: list에 하나의 요소가 남았으므로 비교없이 결과 배열로 이동합니다.
        -> list = [], result = [2, 9, 17, 35, 85]

[step 1을 조금 자세히 살펴보겠습니다.]

step 1-0: index 에 기본값을 설정합니다. -> [index = 0]

step 1-1: (index가 0인 위치에 있는) 35보다 9가 더 작습니까? -> 예
          -> 9의 index로 변경합니다. [index = 1]

step 1-2: (index가 1인 위치에 있는) 9보다 2가 더 작습니까? -> 예
          -> 2의 index로 변경합니다. [index = 2]

step 1-3: (index가 2인 위치에 있는) 2보다 85가 더 작습니까? -> 아니오
          -> 2의 index를 유지합니다. [index = 2]

step 1-4: (index가 2인 위치에 있는) 2보다 17이 더 작습니까? -> 아니오
          -> 2의 index를 유지합니다. [index = 2]

list의 모든 요소의 비교를 마친 시점에서 index가 2인 위치에 있는 2가 가장 작은 수로, 이것을 결과 배열인 result로 이동합니다. 따라서,
          -> list = [35, 9, 85, 17], result = [2]

step 2부터 step 5까지 같은 과정으로 수행하면 최종적으로 list의 숫자들이 작은 수부터 큰 수 순서대로 정렬됩니다.

Time Complexity (복잡도)

리스트의 첫 요소부터 마지막 요소까지 순차적으로, 해당 요소와 해당 요소를 제외한 나머지 요소를 비교하는 방법으로 구성되므로 복잡도는 O(n²)으로 나타낼 수 있겠습니다.

한 문제 더!!

아래와 같은 리스트를 작은 수부터 큰 수로 배열한다고 가정하자.

list = [11, 7, 12, 14, 19, 1, 6, 18, 8, 20];

다음 중 세 번째 순서를 마쳤을 때의 리스트 형태를 나타낸 것은 어떤 것인가?

1) [1, 11, 12, 14, 19, 6, 7, 18, 8, 20]
2) [1, 6, 7, 14, 19, 11, 12, 18, 8, 20]
3) [1, 6, 11, 14, 19, 7, 12, 18, 8, 20]
4) [1, 6, 11, 12, 19, 7, 14, 18, 8, 20]

Answer: 2)
