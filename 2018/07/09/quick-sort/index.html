Quick Sort (퀵 정렬)

퀵 정렬(Quick Sort)은 ‘그룹을 둘로 나누어 재귀 호출을 이용하여 정렬’한다는 면에서는 병합 정렬(Merge Sort)과 같습니다.
그러나, 병합 정렬(Merge Sort)은 그룹을 임의의 둘로 나누는 데 반해 퀵 정렬(Quick Sort)은 그룹을 기준점과 비교해서 나눈다는 것이 병합 정렬과의 다른 점이라고 할 수 있습니다.
즉, 퀵 정렬(Quick Sort)은 ‘기준과 비교해서 그룹을 나누고 재귀 호출을 이용하여 정렬’하는 방식입니다.

문제

list = [35, 9, 2, 85, 17];

위의 리스트 안의 자료를 작은 수부터 큰 수 순서로 배열하는 정렬 알고리즘을 만드시오.

[언뜻 생각한 알고리즘]

기준(pivot)이 되는 값을 정합니다. 여기서는 리스트의 가장 마지막 값을 기준값으로 정하겠습니다. (pivot = 17)
리스트의 모든 값을 기준값(pivot)과 비교하여 작은 값은 group1로, 큰 값은 group2로 넣은 후,
작은 값들의 모음인 group1, pivot 값, 큰 값들의 모음인 group2의 값을 순서대로 합하면 정렬된 리스트를 구할 수 있습니다.
group1과 group2는 각각을 다시 재귀적으로 정렬합니다.

step 1: 35가 17보다 작습니까? -> 아니오 -> group2에 값을 넣습니다.
      : group1 = [], group2 = [35]

step 2: 9가 17보다 작습니까? -> 예 -> group1에 값을 넣습니다.
      : group1 = [9], group2 = [35]

step 3: 2가 17보다 작습니까? -> 예 -> group1에 값을 넣습니다.
      : group1 = [9, 2], group2 = [35]

step 4: 85가 17보다 작습니까? -> 아니오 -> group2에 값을 넣습니다.
      : group1 = [9, 2], group2 = [35, 85]

…
step n: 정렬을 마친 group1과 pivot 값인 17, 정렬을 마친 group2를 합합니다.

* group1과 group2는 각각 다시 퀵 정렬(quick sort)을 수행합니다.

Code (in Javascript) #1

이를 코드로 구현해 보면 아래와 같습니다.

[조금 더 보편적인 알고리즘]

이를 조금 더 보편적인 알고리즘으로 생각해 보면 아래와 같이 설명할 수 있습니다.
새로운 두 그룹으로 나누고 그것을 다시 합치는 과정을 없애고, 그냥 같은 리스트 안에서 자리를 서로 바꾸는 방법입니다.

기준값과 리스트의 나머지 값들을 비교합니다.
비교할 값들 중 가장 왼편의 index를 ‘leftmark’, 가장 오른편의 index를 ‘rightmark’라고 합니다.

leftmark를 하나씩 증가하며 값을 비교, 기준값(pivot)보다 큰 값을 만나면 비교를 중단합니다. (*1)
rightmark를 하나씩 감소하며 값을 비교, 기준값(pivot)보다 작은 값을 만나면 비교를 중답합니다. (*2)

앞서의 두 수 (*1)과 (*2)의 자리를 서로 바꿉니다.

앞선 방법으로 다시 탐색을 하여 leftmark가 rightmark와 같거나 크게 되면 탐색 과정을 중단하고 그에 해당하는 값과 기준값(pivot)의 자리를 바꿉니다.

기준값의 앞과 뒤는 재귀적으로 위의 과정과 동일한 과정으로 정렬합니다.

기준값은 앞서와 마찬가지로 리스트의 가장 마지막 값으로 정하겠습니다. (pivot = 17)
leftmark가 rightmark 보다 작다는 조건은 기본적으로 염두에 두겠습니다.

* 왼쪽을 시작으로 큰 값을 만날 때까지 계속 탐색
step 1: 35가 17보다 큽니까? -> 예 -> stop (*1) : leftmark = 0, rightmark = 3

* 오른쪽을 시작으로 작은 값을 만날 때까지 계속 탐색
step 2: 85는 17보다 작습니까? -> 아니오 -> 계속 탐색 : leftmark = 0, rightmark = 2
step 3: 2는 17보다 작습니까? -> 예 -> stop (*2) : leftmark = 0, rightmark = 2

* 두 수의 자리를 바꿈
step 4: (*1)과 (*2)의 자리를 바꾸고 계속 탐색 : leftmark = 1, rightmark = 2

* 왼쪽을 시작으로 큰 값을 만날 때까지 탐색 계속
step 5: 9는 17보다 큽니까? -> 아니오 -> 계속 탐색 : leftmark = 2, rightmark = 2
step 6: 35는 17보다 큽니까? -> 예 -> stop (*3) : leftmark = 2, rightmark = 2

* leftmark와 rightmark가 같게됨
step 7: (*3)과 pivot의 자리를 바꾸고 탐색 중단

Time Complexity (복잡도)

퀵 정렬(quick sort)도 분할 정복(Divide and Conquer)을 기반으로 하므로 계산 복잡도는 O(n·log n)이 됩니다.
