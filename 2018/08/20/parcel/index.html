ParcelJS

ParcelJS 사이트에 기재되어 있는 Parcel의 장점 중에 첫 번째가 “Blazing fast bundle times”인데 의미는 알겠으나 한국어로는 어떻게 번역할 수 있는가를 고민하다가 가만 보니 ParcelJS에서 한국어를 지원하고 있음을 알았다.

  • Parcel : https://parceljs.org/
  • Parcel 한국어판 : https://ko.parceljs.org/

Parcel 한국어 사이트에 그 특징이 잘 나와있다.

  • Parcel의 특징
  • 불꽃 튀게 빠른 번들 시간 (Blazing fast bundle times)
  • 모든 자원의 번들 (Bundle all your assets)
  • 자동 변환 (Automatic transforms)
  • 설정없는 코드 분할 (Zero config code splitting)
  • 빠른 모듈 교체 (Hot module replacement)
  • 친절한 에러 로그 (Friendly error logging)

가장 큰 특징은 “설정없이, 빠르게”로 축약할 수 있을 것 같습니다.

Why Parcel?

Webpack, Broswerify 등 거대한 여러 Web application bundler가 있음에도 왜 또 다른 것이 필요한 걸까요?
다른 여러 번들러들은 다소 복잡한 설정(Configuration)과 플러그인(Plugin)이 필요한데 Parcel은 별도의 설정(Configuration)없이도 사용할 수 있다는 것이 큰 매력인 것 같습니다.
(사실 저는 Webpack나 Broswerify를 사용해 보지 않아 얼마나 복잡한 설정이 필요한 지는 모르겠습니다만, 어쨌든 무엇인가 ‘설정을 해야 한다’는 것은 부담이 있기 마련입니다.)

또한, 덩치가 큰 application의 경우는 다수의 파일과 그 파일들에 관련된 Dependency를 모두 체크하려면 시간이 꽤 소요되는데, 특히나 개발 중 변경 사항이 많은 경우 빌드 시간을 기다리는 것은 아주 번거로운 일이 아닐 수 없습니다.
파일 Watcher가 재빌드해야 하는 상태를 계속 주시하고 있다고 하더라도 초기 빌드는 그래도 시간이 걸리게 마련입니다.
Parcel은 현대 멀티 코어 프로세서를 사용하여 코드를 병렬적(parallel)으로 컴파일하기 때문에, 특히 초기 빌드 시에 아주 빠른 결과를 얻을 수 있습니다.
